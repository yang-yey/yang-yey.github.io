记录 JAVA SE 学习过程中的一些基础知识点
以防遗忘基础

基础语法

1、数据类型

**bit(位)**： 0/1
**Byte(字节)**： 8位，数据存储的最小单位

数据类型 关键字 内存占用 取值范围
字节型 byte 1(字节) -128 - 127
短整型 short 2 -32768 - 32767
整型(默认) int 4 - 2^31^ - 2^31^ -1
长整型 long 8 - 2^63^ - 2^63^ -1
单精度浮点数 float 4 1.4013E-45 - 3.4028E+38
双精度浮点数（默认） double 8 4.9E-324 - 1.7977E+308
字符型 char 2 0 - 65535
布尔型 boolean 1 true、false

注 :
1、字符串是引用类型；
2、浮点型是近似值；
3、创建float和long类型时，字母后缀F、L勿忘
4、创建byte和short类型时，务必不能超过取值范围
5、浮点数强转为int，会舍弃小数位

2、进制换算

十进制–>二进制 除以2取余，倒序
二进制–>十进制 2^n^ 相加

3、运算符

算数运算符： + - * / % ++ –
赋值运算符： = += -= *= /= %=
比较运算符： == < > <= >= !=
逻辑运算符： && || ！
三元运算符 : c = a>b ? a:b;

4、流程控制语句

if-(else) switch-(default) for (do)-while **
死循环：while(true){ 循环体 }
**注 ： switch小括号里 只能是 char byte short int 或 String enum
or Character Byte Short Integer

5、数组

eg : int[] arr = {1,3,5};
初始化默认值：
整型：0 浮点型：0.0 字符型：’\u0000’ 布尔型： false 引用类型：null
长度：数组名称.length;
排序：Arrays.sort(数组名称)；升序

面向对象（封装&继承&多态）

编程思想:
面向过程->面向对象->面向接口->面向组件->面向切面->面向服务

1、 对OOP的理解

  • 1、 首先面向对象是一种编程思想

  • 2、 万物皆对象。OOP可以理解为使用代码来模拟现实生活。

  • 3、 三大特性：封装、继承、多态
    封装：就是隐藏类的内部信息，不允许外部程序直接访问，而是通过getter 和 setter 方法完成，提高数据安全性。
    继承是指：父类的基本特征和行为，子类也会有，同时子类也能修改这些特征和行为。

    多态：同一行为具有多个不同的表现形式或形态的能力。

2、类

标准类构成：
成员变量(private) 构造方法(无参构造 全参构造) 成员方法(getter & setter)

构造方法
public 类名称 (参数类型 参数名称){方法体}
构造方法无返回值 ，可重载overload（方法名相同，参数列表不同）

成员方法(getter & setter)
对于基本类型的Boolean值，getter方法一定要写成isXxx的形式

类的使用

  • 创建类对象
    单个对象：类名 对象名=new 类名(); 

        Person one =  new Person();

    对象数组：类名 [] 数组名=new 类名[长度];

        Person[] arr = new Person[3];

  • 参数传递和返回

静态代码块
static{ } 当且仅当类加载时执行一次
对象初始化顺序：父类静态代码块 > > 子类静态代码块 > > 父类非静态代码块 > > 父类构造方法 > > 子类非静态代码块 > > 子类构造方法

3、继承 （extends）
3.1成员变量&成员方法

成员变量：
1、成员变量不重名：无影响
2、成员变量重名：
创建子类对象时，访问变量：
i.直接通过子类对象访问成员变量： 等号左边是谁，优先用谁，没有向上找
i.间接通过成员方法访问成员变量： 方法属于谁，优先用谁，没有向上找

局部变量： 直接写成员变量名
本类的成员变量： this.成员变量名
父类的成员变量： super.成员变量名
父类变量若为私有，使用需要用 getter、setter

成员方法：
1、成员方法不重名：无影响
2、成员方法重名（重写override）：使用子类重写后的方法
子类若想调用父类未重写的方法，需先向上转型（参考多态处）

注：
子类方法覆盖父类方法，要保证【权限】>=父类权限修饰符，【返回值】<=父类返回值范围
public > protected > (default) > private

3.2构造方法

i. 子类是无法继承父类构造方法（构造方法的名字是与类名一致）
i. 子类的构造方法中默认有一个 super() ，表示调用父类的构造方法，父类成员变量初始化后，才可以给子类使用。
i.只有子类构造方法才能调用父类构造方法。 public Zi(){ super(); } 详见下

注：
1、每次创建子类对象时，先初始化父类空间，再创建其子类对象本身。
2、子类的每个构造方法中均有默认的super()，调用父类的空参构造。手动调用父类构造会覆盖默认的super()。 super() 和 this() 都必须是在构造方法的第一行，所以不能同时出现

  • super关键字的3种用法
    1、在子类的成员方法中，访问父类的成员变量
    2、在子类的成员方法中，访问父类的成员方法
    3、在子类的构造方法中，访问父类的构造方法

  • this关键字的3种用法
    1、在本类的成员方法中，访问本类的成员变量
    2、在本类的成员方法中，访问本类的另一个成员方法
    3、在本类的构造方法中，访问本类的另一个构造方法(唯一且第一个语句)

3.3抽象方法(abstract)

抽象方法 ： 没有方法体的方法。 修饰符 abstract 返回值类型 方法名 (参数列表)；
抽象类：包含抽象方法的类。 abstract class 类名字 { }

3.4接口(implements)

接口可包含：

1、成员变量其实是常量，格式：
[public] [static] [final] 数据类型 常量名称 = 数据值; int NUM = 1；
注意：
常量必须进行赋值，而且一旦赋值不能改变。
常量名称完全大写，用下划线进行分隔。

2、方法1——抽象方法，格式：
[public] [abstract] 返回值类型 方法名称(参数列表);
注意：实现类必须覆盖重写接口所有的抽象方法，除非实现类是抽象类。

3、方法2——默认方法，格式：
[public] default 返回值类型 方法名称(参数列表) { 方法体 }
注意：默认方法也可以被覆盖重写
应用：接口升级，修改默认方法，能被实现类继承

4.、方法3——静态方法，格式：
[public] static 返回值类型 方法名称(参数列表) { 方法体 }
注意：应该通过接口名称进行调用，不能通过实现类对象调用接口静态方法

5、从Java 9开始，方法4——私有方法，格式：
普通私有方法：private 返回值类型 方法名称(参数列表) { 方法体 }
——供多个默认方法使用
静态私有方法：private static 返回值类型 方法名称(参数列表) { 方法体 }
——供多个静态方法使用
private的方法只有接口自己才能调用，不能被实现类或别人使用

  • 接口中，没有构造方法，不能创建对象。
  • 接口中，没有静态代码块

注：
1、一个类，继承一个父类、若干个接口，若父类中的成员方法与接口中的默认方法重名，子类就近选择执行父类的成员方法。
2、一个接口如果多个父接口中的默认方法有重名的，那么子接口需要重写一次该方法
子接口重写默认方法时，default关键字可以保留。
子类重写默认方法时，default关键字不可以保留。
